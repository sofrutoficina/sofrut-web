# Política de Seguridad

## Reportar una Vulnerabilidad

Si descubres una vulnerabilidad de seguridad en este proyecto, por favor repórtala de manera responsable.

### Cómo Reportar

Por favor **NO** abras un issue público en GitHub.

En su lugar, contacta directamente al equipo de desarrollo a través de:
- **Email:** sofrutoficina@gmail.com

### Información a Incluir

Al reportar una vulnerabilidad, incluye:
- Descripción detallada del problema
- Pasos para reproducir la vulnerabilidad
- Impacto potencial
- Sugerencias de corrección (si las tienes)

### Proceso de Respuesta

1. Confirmaremos la recepción en **48 horas**
2. Evaluaremos la vulnerabilidad
3. Te mantendremos informado del progreso
4. Publicaremos una corrección lo antes posible

## Mejores Prácticas para Usuarios

### Entorno de Desarrollo
- Mantén las reglas de Firestore en modo restrictivo antes de producción
- No compartas credenciales de Firebase
- Mantén `.env.local` fuera del control de versiones

### Producción
- Asegúrate de configurar reglas de seguridad de Firestore
- Usa contraseñas fuertes (mínimo 8 caracteres)
- Revisa periódicamente los logs de acceso

## Agradecimientos

Agradecemos a todos los que reportan vulnerabilidades de manera responsable y ayudan a mantener este proyecto seguro.
